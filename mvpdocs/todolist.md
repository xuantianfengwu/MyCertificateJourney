#### 1.1.1 具体动作

**配置云开发环境**：

1. 在微信开发者工具中点击左侧的"云开发"按钮
2. getUserStudyExams点击"创建环境"，输入环境名称（如：`my-certificate-journey`）
3. 选择免费套餐（适合MVP阶段）
4. 等待环境创建完成
5. 在小程序代码中配置环境ID：
   - 在`app.js`中添加`wx.cloud.init({ env: 'your-env-id' })`

**创建云数据库集合**：

1. 进入云开发控制台，点击左侧"数据库"标签
2. 点击"+ 集合"按钮，创建以下集合，并为每个集合设置适当的权限：

| 集合名称                   | 用途描述           | 权限类型选择                                   |
| ---------------------- | -------------- | ---------------------------------------- |
| `exams`                | 存储考试信息         | Readable by all users                    |
| `users`                | 存储用户信息         | Only the creator can read and write      |
| `studyRecords`         | 存储学习记录         | Only the creator can read and write      |
| `questions`            | 存储题库           | Readable by all users                    |
| `userQuestions`        | 存储用户题目（错题和收藏题） | Only the creator can read and write      |
| `forumPosts`           | 存储论坛帖子         | Readable to all users and readable to be |
| `studySpaces`          | 存储备考空间         | Readable by all users                    |
| `sectMasters`          | 存储盟主信息         | Readable by all users                    |
| `userMasterChallenges` | 存储用户盟主挑战记录     | Only the creator can read and write      |
| `userExamStats`        | 存储用户考试科目统计     | Only the creator can read and write      |

权限说明：

- **Readable by all users**：所有用户可读（用于公开数据）
- **Only the creator can read and write**：仅创建者可读可写（用于用户私有数据）
- **Readable to all users and readable to be**：所有用户可读，仅创建者可写（用于需要作者权限的内容）

**创建云函数**：

1. 进入云开发控制台，点击左侧"云函数"标签
2. 点击"+ 新建云函数"按钮，创建以下云函数，并为每个云函数配置适当的超时时间（建议3-10秒）和内存：
   - `examRecommend`：智能推荐考试
     - **输入参数**：userId（用户ID）、profession（职业）、interests（兴趣）、studyTime（学习时间）
     - **输出结果**：推荐的考试列表，按照匹配度排序
     - **主要功能**：
       - 基于用户职业背景，筛选相关行业的资格考试
       - 根据用户兴趣偏好，推荐适合的考试类型
       - 考虑用户可用学习时间，推荐难度适中的考试
       - 优先推荐可个税抵扣的资格考试
       - 综合计算考试适配度（简单/困难/未知）
       - 支持按通过率降序或费用升序排序
   - `questionBank`：题库管理
     - **输入参数**：examId（考试ID）、subject（科目）、type（题目类型）、count（题目数量）、difficulty（难度）、operation（操作类型：get/submit/check）
     - **输出结果**：题目列表、答案、判题结果、分数统计
     - **主要功能**：
       - 按考试、科目、类型、难度随机获取题目
       - 生成章节练习题目
       - 生成模拟考试试卷（符合考试时长和题型分布）
       - 自动判题并计算得分
       - 将用户错题记录到userQuestions集合（type=error）
       - 将用户收藏题记录到userQuestions集合（type=favorite）
       - 标记错题是否已掌握（mastered字段）
       - 统计题目正确率和完成率
       - 提供错题重练和收藏题回顾功能
   - `userProgress`：学习进度管理
     - **输入参数**：userId（用户ID）、examId（考试ID）、operation（操作类型：update/progress/achievement/masterChallenge）
     - **输出结果**：更新后的学习进度、成就状态、门派称号、盟主挑战状态
     - **主要功能**：
       - 记录用户学习时长和完成的知识点
       - 计算连续学习天数（修炼天数）
       - 更新门派称号（初出茅庐/渐入佳境/小有成就/炉火纯青/登峰造极/一代宗师）
       - 生成每日修炼任务（门派教材）
       - 管理退出师门逻辑（清空当前门派修炼进度）
       - 处理叩谢师门请求（提交考试通过图片）
       - 统计门派弟子排名
       - 计算修炼进度百分比
       - 验证是否达到一代宗师条件
       - **盟主挑战相关功能**：
         - 获取当前科目盟主列表和用户挑战状态
         - 开始盟主挑战（生成挑战题目）
         - 提交盟主挑战结果
         - 解锁盟主私房照和视频
         - 统计用户总挑战次数和胜率
         - 统计每个盟主的总挑战次数和胜率
         - 检查是否完成当前盟主7次挑战，解锁下一个盟主
   - `studyRecord`：学习记录管理
     - **输入参数**：userId（用户ID）、examId（考试ID）、date（日期）、operation（操作类型：add/get/statistics）
     - **输出结果**：学习记录、统计数据、排行榜
     - **主要功能**：
       - 添加每日学习记录
       - 获取用户历史学习记录
       - 统计总学习时长
       - 统计学习天数
       - 计算日均学习时间
       - 生成学习周报/月报
       - 更新用户在门派中的学习排名
       - 记录和更新用户成就信息
       - 统计盟主挑战次数和胜率
   - `forumPosts`：论坛帖子管理
     - **输入参数**：userId（用户ID）、city（城市）、operation（操作类型：get/add/like/comment）
     - **输出结果**：帖子列表、发布结果、点赞状态、评论列表
     - **主要功能**：
       - 获取指定城市的江湖留言
       - 发布新的江湖留言
       - 处理帖子点赞功能
       - 处理帖子评论功能
       - 获取备考空间列表（按城市筛选）
       - 统计帖子的点赞数和评论数
       - 按时间或热度排序帖子
       - 验证用户是否可以发布留言（防刷屏机制）

**配置云存储**：

1. 进入云开发控制台，点击左侧"存储"标签
2. 点击"+ 新建文件夹"按钮，创建以下文件夹：
   - `materials`：存储考试资料
   - `userUploads`：存储用户上传内容
   - `sectMasters`：存储盟主资料
     - 在 `sectMasters` 文件夹下为每个考试科目创建子文件夹
     - 文件命名规范（直接放在考试科目文件夹下）：
       - `{masterId}_1.jpg`：盟主第1张私房照
       - `{masterId}_2.jpg`：盟主第2张私房照
       - ...
       - `{masterId}_7.jpg`：盟主第7张私房照
       - `{masterId}_video.mp4`：盟主私房视频
3. **设置权限（只能统一设置）**：
   - 选择"All users can read"（所有用户可读）
   - 说明：微信云存储权限只能统一设置，无法针对不同文件夹设置不同权限
   - 私密内容的权限控制在业务逻辑层面实现（例如：用户上传内容在代码中验证用户身份，盟主内容通过用户解锁状态判断）
4. 配置存储安全规则，确保数据安全
